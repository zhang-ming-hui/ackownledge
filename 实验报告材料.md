# 信息检索与信息抽取实验报告材料

## 1. 项目概述

本项目围绕 skills 数据集实现了两个可独立运行、又可以联动的实验系统：

- `ir_system`：信息检索系统
- `ie_system`：信息抽取系统

数据来自开源网页抓取结果，当前本地样本规模为 1000 条 skill 文档，满足不少于 100 篇文档的实验要求。

## 2. 系统组成

### 2.1 信息检索系统

信息检索系统实现了：

- 本地 JSON 数据存储
- 倒排索引构建
- TF-IDF 向量空间检索
- BM25 词项加权检索
- Hybrid 混合检索
- 中英混合查询扩展
- 失败样例桶分析
- Web 人工相关度评价

### 2.2 信息抽取系统

信息抽取系统从每条 skill 描述中抽取 6 类信息点：

- `platforms`
- `languages`
- `action_types`
- `target_domains`
- `output_formats`
- `metrics`

这些信息点可组合成一个“技能事件”，例如：

- 谁：`skill_name`
- 在什么平台：`platforms`
- 使用什么技术：`languages`
- 执行什么动作：`action_types`
- 面向什么领域：`target_domains`
- 输出什么格式：`output_formats`
- 规模指标是什么：`metrics`

## 3. 基础算法设计

### 3.1 IR 基础算法

- 文档分词：英文按单词与词形变体切分，中文按字串、单字、双字切分
- 倒排索引：对 `skill_name / description / category / owner / repo` 建立倒排索引
- 向量空间模型：使用 TF-IDF + cosine similarity
- 词项检索模型：使用 BM25

### 3.2 IE 基础算法

- 关键词词典匹配
- 正则表达式匹配
- 数量模式抽取
- 规则增强抽取

## 4. 第 5 档创新点

本项目的创新点不是停留在说明层，而是落实在系统与实验流程中：

### 创新点 1：查询感知的 Hybrid 检索

不是固定使用单一检索算法，而是根据 query 特征动态控制 BM25 在混合排序中的占比：

- 中文 query：降低 BM25 权重，减少词面噪声干扰
- 英文短 query：保留一定 BM25 贡献，提高精确词命中能力
- 结果中同时暴露 `cosine_score` 和 `bm25_score`，增强可解释性

### 创新点 2：模式对比可复现

信息检索系统新增了模式对比入口，可以直接对比：

- `tfidf`
- `bm25`
- `hybrid`

并输出：

- Hit@K
- Top1 Accuracy
- MRR@K
- Recall@K
- Precision@K
- 失败样例摘要

### 创新点 3：信息抽取的增强规则消融对比

信息抽取系统支持两种变体：

- `baseline`：基础词典/正则抽取
- `enhanced`：加入动作别名扩展规则后的增强抽取

这样就可以直接做消融实验，说明优化前后差异，而不是只展示最终版本。

### 创新点 4：检索与抽取联动思路

本项目已经具备第 5 档进一步扩展的结构基础：

- 先用 IR 缩小候选 skill 集
- 再对候选集执行 IE
- 比较“全量抽取”和“检索后抽取”的效率与准确率

该方向适合继续作为论文式创新扩展。

## 5. 算法对比实验

### 5.1 IR 当前结果

当前 `core_relevance.json` 上的 Hybrid 结果：

- `Hit@5 = 100.00%`
- `Top1 Accuracy = 85.71%`
- `MRR@5 = 0.9087`
- `Recall@5 = 93.45%`
- `Precision@5 = 27.62%`

当前仍有部分 `not_top1` 与 `partial_expected_coverage` 样例，说明系统还有优化空间，但整体已经满足课程实验需求，并具备进一步分析的基础。

### 5.2 IE 当前结果

当前 `ground_truth.json` 上增强版抽取结果：

- `Precision = 57.77%`
- `Recall = 62.50%`
- `F1 = 60.04%`

其中动作字段 `action_types` 相比之前已有明显改善，说明增强规则是有效的。

### 5.3 推荐展示方式

实验报告中建议至少放两张表：

1. IR 模式对比表
   - TF-IDF / BM25 / Hybrid 的五个指标横向对比

2. IE 变体对比表
   - Baseline / Enhanced 的总体 Precision / Recall / F1 对比
   - 各字段 F1 对比

## 6. 人工评价设计

### 6.1 IR 人工评价

对检索结果的 Top-K 文档进行人工标注：

- `relevant`
- `partial`
- `not_relevant`

系统会汇总人工评价统计，形成查询级和总体级精度指标。

### 6.2 IE 人工评价

对抽取字段进行人工评价：

- `correct`
- `partial`
- `incorrect`

并统计：

- 总体准确率
- 按字段准确率

## 7. 可持续性与社会影响

课程要求中提到环境和社会可持续发展影响，本项目可从以下角度说明：

- 本地索引与本地抽取减少重复联网调用，降低资源消耗
- 通过失败样例分析和模式对比，避免盲目扩大模型与算力开销
- 系统强调人工评价与可解释性，减少错误自动化传播
- 对信息检索与抽取结果保留人工复核接口，降低误导性输出风险

## 8. 复现实验命令

### 8.1 IR

```bash
cd ir_system
python -m src.skills_ir build-index --force
python -m src.skills_ir evaluate
python -m src.skills_ir compare-modes --eval-set core_relevance.json --eval-set multilingual.json --top-k 5
python -m src.skills_ir review-system --top-k 5
```

### 8.2 IE

```bash
cd ie_system
python -m src.skills_ie extract --variant enhanced
python -m src.skills_ie evaluate --variant enhanced
python -m src.skills_ie compare --eval-set ground_truth.json
python -m src.skills_ie report --variant enhanced
```

## 9. 仍可继续优化的方向

- IR：进一步压低 `not_top1` 失败样例数量
- IR：引入 reranker 或更细粒度 query intent 分类
- IE：继续增强 `target_domains` 和 `output_formats` 的规则
- IE：加入字段级抽取来源追踪，提高可解释性
- IR + IE：实现“检索后抽取”的联动实验
