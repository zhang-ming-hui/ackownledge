Iterative Retrieval Pattern Solves the "context problem" in multi-agent workflows where subagents don't know what context they need until they start working. When to Activate Spawning subagents that need codebase context they cannot predict upfront Building multi-agent workflows where context is progressively refined Encountering "context too large" or "missing context" failures in agent tasks Designing RAG-like retrieval pipelines for code exploration Optimizing token usage in agent orchestration The Problem Subagents are spawned with limited context.
