# Skills 信息检索系统说明

## 1. 项目目标

基于本地数据集 `../data/skills_data.json` 构建一个面向技能发现的检索系统。  
用户输入自然语言查询后，系统返回最相关的 skill，并输出：

- 相关度分数
- skill 名称
- 类别、作者、仓库
- 匹配摘要
- 详情 URL
- 首次收录时间

## 2. 数据来源

- 数据文件：`../data/skills_data.json`
- 文档数量：500 条
- 每条文档主要字段：
  - `skill_id`
  - `skill_name`
  - `owner`
  - `repo`
  - `description`
  - `category`
  - `detail_url`
  - `first_seen`
  - `weekly_installs_num`
  - `github_stars_num`

## 3. 系统设计

### 3.1 检索模型

系统采用“两阶段检索”：

1. 倒排索引召回
   - 对 `skill_name / description / category / owner / repo` 建立倒排索引
   - 先根据查询词快速召回候选文档

2. 向量空间排序
   - 使用 TF-IDF 计算查询和文档的向量表示
   - 采用余弦相似度进行排序

### 3.2 查询预处理

为了让中文查询也能匹配英文描述，系统做了中英混合处理：

- 英文：按单词切分
- 中文：按词串、单字、双字片段切分
- 查询扩展：将常见中文意图词映射到英文检索词
  - 例如：`前端 -> frontend / ui / react / css`
  - 例如：`爬虫 -> scrape / crawl / browser`
  - 例如：`接口 -> api / rest / backend`

### 3.3 排序增强

在 TF-IDF 余弦相似度基础上，系统增加了两类轻量加权：

- 标题精确命中加分
- 热度加分
  - `weekly_installs_num`
  - `github_stars_num`

这样能让语义匹配和实际流行度同时影响结果排序。

## 4. 运行方式

### 4.1 单次查询

```bash
python .\skills_ir_system.py "api design"
python .\skills_ir_system.py "前端页面设计"
python .\skills_ir_system.py "爬虫 自动化" --top-k 10
```

### 4.2 交互式查询

```bash
python .\skills_ir_system.py --interactive
```

### 4.3 重建索引

```bash
python .\skills_ir_system.py --rebuild "api design"
```

索引文件会保存为本地文件：

- `runtime/skills_ir_index.json`

### 4.4 运行评测

```bash
python .\skills_ir_system.py --evaluate --top-k 5
```

评测集文件：

- `eval/skills_eval_queries.json`

输出指标包括：

- `Hit@K`
- `MRR@K`

## 5. 当前实现特点

- 不依赖外部数据库，直接基于 JSON 文件构建本地检索
- 已满足“本地存储 + 倒排索引 + 向量空间模型排序”的基本要求
- 能处理英文查询，也能处理一部分中文意图查询
- 支持结果摘要展示，方便人工评估检索准确率

## 6. 后续可优化方向

- 增加中文同义词词典，提升中文查询召回率
- 引入 BM25 替代 TF-IDF，进一步提升排序质量
- 加入 Web 页面或可视化界面
- 增加人工标注评测集，做更正式的准确率、召回率、NDCG 评估
- 支持按类别、热度、时间做过滤检索
